1. What is DevOps?

DevOps is a culture and a set of practices that brings Development and Operations together to deliver software faster, more reliably, and with greater collaboration.



2. Why was DevOps created?

DevOps was created to bridge the gap between the Development and Operations teams. Before DevOps, developers and operations often worked separately, resulting in deployment delays, environment inconsistencies, and applications that worked during development but failed in production.



3. What problems does it solve?

DevOps solves several common problems such as:

- Slow and manual deployments
- Lack of collaboration between Development and Operations teams
- Inconsistent environments between development, testing, and production
- Repetitive manual tasks through automation
- Faster recovery when failures occur



4. What does a DevOps Engineer do?

DevOps Engineer solve problems like:
* How do we deploy software safely?
* How do we automate repetitive work?
* How do we keep system running?
* How do we recover quickly if something breaks?
* How do we make deployments consistent?
* How do we monitor applications?
* How do we secure deployments?
