# GitHub Actions Status Badge

## Objective

Learn how GitHub Actions workflow status badges provide real-time build visibility, allowing developers and visitors to quickly determine whether the latest Continuous Integration (CI) workflow has passed or failed.

---

## What is a Workflow Status Badge?

A workflow status badge is a dynamic image generated by GitHub that displays the current status of a workflow.

The badge automatically updates whenever the workflow runs and reflects the latest result for a selected branch.

Typical statuses include:

- Passing
- Failing
- In Progress

---

## Why Use Status Badges?

Reasons for using workflow status badges:

1. Build Visibility: Instantly shows whether the latest build succeeded or failed.
2. Repository Health: Provides a quick indication of project stability.
3. Professional Appearance: Commonly used in open-source and production repositories.
4. Easy Navigation: The badge links directly to the workflow execution history.

---

## How Workflow Status Badges Work

Workflow execution:

```text
Developer Pushes Code
        │
        ▼
GitHub Actions Starts
        │
        ▼
Workflow Executes
        │
        ▼
Workflow Result
        │
        ├──────────────┐
        │              │
      Success        Failure
        │              │
        ▼              ▼
 Green Badge      Red Badge
        │
        ▼
README Updates Automatically
```

---

## Badge URL Structure

A workflow badge is generated using the workflow file name.

General format:

```text
https://github.com/<owner>/<repository>/actions/workflows/<workflow>.yml/badge.svg
```

Example from this repository:

```text
https://github.com/rhanifrizal/learning-devops/actions/workflows/ci.yml/badge.svg
```

To display the status for a specific branch:

```text
https://github.com/rhanifrizal/learning-devops/actions/workflows/ci.yml/badge.svg?branch=main
```

## Branch-Specific Status

Workflow badges can be configured to display the status of a specific branch.

Example:

```text
?branch=main
```

This ensures the badge always reflects the health of the production branch rather than temporary feature branches.

## Interview Questions

### What is a GitHub Actions status badge?

A workflow status badge is a dynamic image that displays the latest execution status of a GitHub Actions workflow.

---

### Why are workflow badges useful?

They provide immediate visibility into the health of a project's Continuous Integration pipeline.

---

### Where can workflow badges be displayed?

Workflow badges are commonly placed in the repository README and can also be shared in documentation or project websites.

---

### What happens when a workflow fails?

The badge automatically changes from **Passing** to **Failing**, indicating that the latest workflow execution was unsuccessful.

---

### Why specify a branch in the badge URL?

Specifying a branch ensures the badge reflects the workflow status of a particular branch, such as `main`, instead of all branches.

---

## Common Mistakes

- Forgetting to use the correct workflow filename.
- Referencing the wrong repository owner or repository name.
- Omitting the branch parameter when monitoring the main branch.
- Assuming static Shields.io badges update automatically.
- Expecting the badge to update before the workflow completes.
